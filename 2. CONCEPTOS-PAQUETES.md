## ¿Qué son paquetes en Java?

Cuando hacemos programas más grandes, es normal que el número de clases vaya creciendo. Meterlas todas en el mismo directorio no suele ser una buena práctica.

Los **paquetes** permiten **agrupar clases relacionadas**, facilitando:

- Organización del código
- Reutilización
- Mantenimiento
- Evitar conflictos de nombres

**Ejemplo:**

- Clases de base de datos → `datos`
- Clases de interfaz gráfica → `ui`
- Clases de impresión → `impresion`

 Un paquete es simplemente un **grupo lógico de clases relacionadas**.

---

## La clase HolaMundo en un paquete

Para indicar que una clase pertenece a un paquete, se usa la palabra clave `package` **en la primera línea** del archivo `.java`.

```java
package prueba;

public class HolaMundo {
    public static void main(String[] args) {
        System.out.println("Hola mundo");
    }
}
```

El código es igual al Hola Mundo tradicional, pero ahora la clase pertenece al paquete `prueba`.

## Estructura de directorios asociada

Java **obliga** a que la estructura de directorios coincida con el nombre del paquete.

### Linux / macOS

```text
/<directorio_usuario>/mi_proyecto_HolaMundo/prueba/HolaMundo.java
```

### Windows

```text
C:\<directorio_usuario>\mi_proyecto_HolaMundo\prueba\HolaMundo.java
```

⚠️ Las **mayúsculas y minúsculas importan** y deben coincidir exactamente con el nombre del paquete.

---

## Compilar una clase que está en un paquete

Debemos posicionarnos en el **directorio padre del paquete**.

### Linux / macOS

```bash
cd /<directorio_usuario>/mi_proyecto_HolaMundo
javac prueba/HolaMundo.java
```

### Windows

```cmd
cd C:\<directorio_usuario>\mi_proyecto_HolaMundo
javac prueba\HolaMundo.java
```

Si todo va bien, se generará:

```text
prueba/HolaMundo.class
```

---

## Ejecutar una clase que está en un paquete

Para ejecutar una clase de un paquete, se usa el **nombre completo**:

```text
paquete.Clase
```

### Linux / macOS

```bash
java prueba.HolaMundo
```

### Windows

```cmd
java prueba.HolaMundo
```

Salida esperada:

```text
Hola Mundo
```

---

## El CLASSPATH cuando las clases están en paquetes

El `CLASSPATH` debe apuntar al **directorio raíz del proyecto**, no al paquete.

### Usando variable de entorno

#### Linux / macOS

```bash
export CLASSPATH=/<directorio_usuario>/mi_proyecto_HolaMundo
java prueba.HolaMundo
```

#### Windows

```cmd
set CLASSPATH=C:\<directorio_usuario>\mi_proyecto_HolaMundo
java prueba.HolaMundo
```

### Usando `-cp`

#### Linux / macOS

```bash
java -cp /<directorio_usuario>/mi_proyecto_HolaMundo prueba.HolaMundo
```

#### Windows

```cmd
java -cp C:\<directorio_usuario>\mi_proyecto_HolaMundo prueba.HolaMundo
```

## Paquetes anidados

Java permite **subpaquetes**, separando niveles con puntos (`.`).

```java
package prueba.basicas;
```

```java
package prueba.avanzadas;
```

### Estructura de directorios

```text
mi_proyecto_HolaMundo/
└── prueba/
    ├── basicas/
    │   └── HolaMundo.java
    └── avanzadas/
        └── HolaMundoRemoto.java
```

### Compilar

```bash
javac prueba/basicas/HolaMundo.java
```

### Ejecutar

```bash
java -cp /<directorio_usuario>/mi_proyecto_HolaMundo prueba.basicas.HolaMundo
```

Salida:

```text
Hola Mundo
```

---

## Resumen

- Los paquetes organizan el código
- El nombre del paquete define la estructura de carpetas
- Se compila desde el directorio raíz del proyecto
- Se ejecuta usando `paquete.Clase`
- El `CLASSPATH` siempre apunta al directorio padre del paquete
